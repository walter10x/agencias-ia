**Placeholders inseguros que siguen en producción (rotar con cuidado, NO tocados aún):**

| Var | Valor actual | Cómo rotar sin romper |
|-----|--------------|------------------------|
| `JWT_SECRET` | `...CHANGE-ME` | `openssl rand -hex 32` → env → redeploy. **Cierra todas las sesiones**. Sin caveat de BD. |
| `POSTGRES_PASSWORD` | `CHANGE_ME_STRONG_PASSWORD_HERE` | La BD **ya se inicializó**: `ALTER USER postgres WITH PASSWORD '<nuevo>';` **primero**, luego igualar el env, luego redeploy. |
| `POSTGREST_AUTHENTICATOR_PASSWORD` | no está en env (usa default) | `ALTER ROLE authenticator WITH PASSWORD '<nuevo>';` **primero**, luego setear el env igual, redeploy. Si no coincide → **PostgREST cae (API 500)**. |
| `WHATSAPP_VERIFY_TOKEN` | `my-verify-token` | Cambiar env → redeploy → **re-verificar** el webhook en Meta (App → WhatsApp → Configuration) con el token nuevo. |
